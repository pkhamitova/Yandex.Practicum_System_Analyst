# Портфолио системного аналитика | Yandex Practicum

Это репозиторий с артефактами, созданными в ходе обучения на курсе «Системный аналитик». Структура репозитория повторяет учебную программу: от базовых понятий до итоговых проектных работ.
## Траектория обучения и артефакты

Каждый модуль содержит материалы, соответствующие пройденной теме. В папках вы найдете документацию, спецификации, схемы, которые я создавала по мере освоения навыков.

| Модуль | Тема | Фокус модуля / Результат |
| :--- | :--- | :--- |
| 01_Understanding_Profession | Понимание профессии | Обоснование выбора каскадной модели (Waterfall) для медицинского ПО: сравнение с альтернативами (Agile, Rolling Wave). Распределение ролей команды по этапам разработки. Перечень артефактов системного аналитика (ТЗ, SRS, требования). |
|02_Requirements_Development | Разработка требований | Шаблоны ТЗ, User Stories, Use Cases, критерии приемки (Acceptance Criteria). |
| 03_Data_Modeling | Моделирование данных | ER-диаграммы, описание сущностей и связей, SQL-скрипты создания таблиц. |
| 04_Process_Modeling | Моделирование процессов | DFD и BPMN диаграммы, описание бизнес-логики. |
| 05_UI_Basic | Пользовательские интерфейсы | Прототипы экранов, описание макетов, сценарии взаимодействия. |
| 06_Agile_Practices | Agile-практики | Примеры планирования спринтов, структура бэклога, метрики. |

## Итоговые проекты

В папке `Projects` собраны комплексные работы, где я объединил навыки из разных модулей для решения одной большой задачи.

*   **Проект 1 (Спецификация требований):** Полный цикл сбора и формализации требований для целевого продукта.
*   **Проект 2 (Моделирование данных):** Проектирование реляционной базы данных, включая нормализацию и оптимизацию запросов.
*   **Проект 3 (Бэклог приложения):** Структурированный бэклог с приоритизацией задач (MoSCoW или RICE) и декомпозицией до уровня задач для разработчика.
